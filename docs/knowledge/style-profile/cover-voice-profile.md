# Cover Voice Profile v1.0

> 这是 **你的小红书账号** 封面生成的 source of truth。
> 所有 RedCard cover 生成的规则、token、约束都以此文档为准。
> 规则更新只动这个文件，代码层无需改动。

---

## §0 适用范围

适用于 RedCard 项目的封面（cover）生成模块。封面是图文笔记 carousel 的第一张，目标：

- 在小红书信息流里 0.5 秒内被识别和点击
- 视觉跟后续卡片同语言，整组观感连贯
- 头像、账号识别、系列标准化

---

## §1 五条铁律（永远不变）

不论选什么 palette、什么 bg 模式、什么内容，封面都遵守：

1. **大色块**：主视觉必有一块占比超 30% 的纯色色块或重色背景
2. **大字**：主标题字号至少占封面短边的 1/6
3. **极简构图**：禁止 3 个以上独立元素堆叠；禁止小标签 / 小图标拼贴；禁止小人物 + 箭头 + 多场景的"营销号"组合
4. **大图大场景**：主视觉素材（人物 / 截图 / 插画）占据封面 50% 以上面积
5. **扎眼吸睛**：颜色对比强烈，文字与背景对比度高，缩略图状态下可识别

## §3 Palette 自适应选择逻辑

模型生成时按以下顺序决定 palette + bg：

### 3.1 优先级
1. 用户手动指定 palette + bg：直接采用，跳过后续推断
2. 用户指定 palette、bg 选 auto：palette 固定，bg 按 3.2 推断
3. 用户全选 auto：palette 和 bg 都按 3.2 推断

### 3.2 推断规则

**Palette family 推断**（根据源素材色调 + 内容性质）：

- 源素材偏温暖 / 杂志感 / 严肃人物访谈 → `red` family
- 源素材偏温暖 / 实战教程 / 工具实测 / 商业话题 → `amber` family
- 源素材偏冷峻 / 学术论文 / 技术内容 / GitHub 项目 → `blue` family
- 源素材偏赛博 / AI 工具 demo / 暗调高对比 → `neon` family（dark only）

**bg 模式推断**：

- 源素材 bg 偏浅、内容偏温和 → `light`
- 源素材 bg 偏深、内容偏议题型 / 反共识 / 新闻 → `dark`
- 内容性质偏教程 / 工具实测 → `dark`（教程类小红书爆款多用深底）
- 无强偏好 → 默认 `light`

### 3.3 冲突解决
- 若推断的 family 是 `neon`，bg 强制 `dark`
- 若用户指定 bg `light` 但推断 family 是 `neon`，回退为 `amber` family（最接近的 light 替代）

## §4 构图原则

封面从上到下大致分三个区域：

### 4.1 顶部主视觉区（约占 60% 面积）
- 主视觉素材（用户上传的人物 / 截图 / 插画）+ 主标题
- 主标题压在主视觉上方或叠加，是第一视觉重心
- 主视觉占据封面 50% 以上面积

### 4.2 中部 accent 区（约占 20% 面积）
- 红色 accent 一句话用色块包裹
- 色块两种处理：深色块 + 反白字，或反色块 + 深字
- 长度严格控制（见 §5.2）

### 4.3 底部区域
- 模型不要生成底部 footer 条、carousel 圆点、页码点、装饰长线、账号头像或账号名
- 底部最多保留一小块安静留白，真实头像和账号名由 Canvas 后处理合成
- 系列标可以出现，但必须极小、克制、像杂志 masthead 的一部分（见 §5.3）

### 4.4 通用构图自由度
- 模型可自由决定：单人/双人/场景为主；主标题位置（顶部/对角/居中）；accent 块位置；字体细节
- 模型不可改变：5 铁律、palette token、底部头像留位、输出尺寸

### 4.5 学语法，不套模板
- 参考图只用于学习风格语法：大字重量、主视觉比例、色块关系、留白密度、杂志/新闻封面的整体气质
- 不允许复刻参考图的固定版式：不要每张都左字右人、不要每张都斜切红块、不要固定底部条、不要固定同一标题位置
- 每次生成都必须根据本篇文章和 source 素材重新组织标题、人物、色块、留白的关系，让画面像为这篇内容单独设计
- 人物如果原图就是西装照、抱胸照、讲师照或商业头像照，可以保留；但必须通过裁切、色块、光影、遮挡或空间关系融入海报，不得变成证件照式抠图贴片
- 禁止廉价 AI 感：塑料皮肤、假棚拍、泛蓝企业科技背景、城市 skyline、PPT 风底部条、孤立小标签、过度平滑、奇怪手或假光影

### 4.6 高级杂志感
- 字体要有编辑部气质：中文优先粗宋 / 标宋 / 书法感衬线，英文用克制窄体 sans 或 editorial serif；避免普通 PPT 黑体、廉价标题字和过度卡通字体
- 色块要像版面结构，不是贴纸：可以用不规则切角、压字、穿插人物、出血边缘、纸张肌理和低饱和高对比色建立整体关系
- 背景要像纸面、摄影棚、真实现场或杂志版面；不要纯渐变科技背景、蓝色企业背景、城市楼群和廉价光效
- 画面需要一个明确视觉重心，标题、人物、色块要互相咬合，不要各自孤立摆放

## §5 文字内容规则

### 5.1 主标题
- 字数：6-15 字，1-2 行
- 字号：极大（占封面短边的 1/6 以上）
- 字体：优先粗宋、标宋、方正粗宋、思源宋粗体等 editorial serif；必要时混少量克制 sans。避免普通 PPT 黑体或廉价电商标题字
- 对比度：必须与背景形成强对比

### 5.2 红色 accent 短句
- 字数：6-10 字，严格控制
- 风格：强观点、不解释、自带钩子
- 示例："7条反共识"、"天道酬勤是最大谎言"、"为什么没有未来"、"不加班的神话"
- 视觉：用 accent 色块包裹

### 5.3 系列标 taxonomy
顶部小标签呈现，可选 5 项：
- `#AI Learning` — 论文共读 / 重要 AI 概念学习
- `#实战现场` — 项目复盘 / 实操记录
- `#搞钱拆解` — AI 商业模式 / 变现路径
- `#论文共读` — 论文精读专题
- `#工具实测` — AI 工具评测

### 5.4 副标题 / 引言（可选）
- 最多 1 行 + 20 字
- 用于补充主标题语义或提供 thesis

### 5.5 禁止
- 超过 3 个独立文字元素
- 小红书话题标签 #xxx 形式（系列标除外）
- emoji
- 多色文字（同一区域内只用 1-2 种颜色）

## §6 头像 + 账号识别（不进 image gen）

**头像永远是后处理贴图，模型不参与生成。**

### 6.1 模型生成时
- 如果用户上传了头像，只在底部左下或右下留一个低矮、安静的小区域，约等于 72px 头像 + 22px 账号名
- 不要绘制头像、头像占位圆、账号名或任何底部署名模块
- 如果用户未上传头像，底部保持干净，不生成账号模块

### 6.2 后处理（Canvas 端）
- 加载用户上传的真实头像 PNG（带 alpha 通道更好）
- 应用圆形 mask
- 合成进预留位置
- 旁边写账号名（取自用户输入的账号名称）

### 6.3 写进 image gen prompt 的指令
> "底部左下或右下只保留一个约 72px 头像 + 22px 账号名的低矮安静区域，不绘制头像、不绘制账号名、不画占位圆，后处理会合成真实账号模块。"

## §7 反例（要避免的风格）

prompt 中显式告知模型避免以下三类：

### 7.1 邪修陷阱
- 特征：亮粉 + 荧光绿乱配、低清自拍感、过度营销号
- 实例：参考 `cover-samples/_avoid/ref01_xiemo.png`
- 问题：与账号的杂志感 / 个人质感定位冲突

### 7.2 国潮 + AI 奇幻陷阱
- 特征：传统中国元素 + AI 生成奇幻角色
- 实例：参考 `cover-samples/_avoid/ref05_xinian.png`
- 问题：跟 AI Learning / 实战内容性质不接

### 7.3 教程拼贴陷阱
- 特征：小人物 + 箭头 + 多场景拼贴 + 多个信息块
- 实例：参考 `cover-samples/_avoid/ref03_ai_ppt.png` / `ref06_dianyinggan.png` / `ref07_4k_pet.png`
- 问题：违反"极简"和"大图大场景"两条铁律

## §8 输出规格

- 尺寸：1080×1440 px（小红书 3:4 竖图标准）
- API 调用尺寸：1056×1408（3:4 竖图，宽高均可被 16 整除，当前默认 image model: gpt-image-2），后处理归一到 1080×1440
- 格式：PNG
- 颜色空间：sRGB
- quality：`high`
- 文件命名：`cover-{articleId}-{paletteFamily}-{bgMode}-{timestamp}.png`
